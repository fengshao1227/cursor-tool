# 更新日志

## v1.0.3 (2025-11-01)

### 🔧 重大修复 - 解决切换账号时会话丢失问题

#### 问题描述
在之前的版本中，切换账号时用户的 Cursor 会话（聊天记录、工作区状态等）会丢失，需要重新开始对话。

#### 根本原因
切换账号时调用的 `factoryReset()` 方法会删除包含会话数据的文件夹（workspaceStorage、History、Session Storage 等），即使之前已经备份，但由于删除操作过于激进导致恢复不完整。

#### 修复方案

**核心改进：**
- ✅ **精细化数据删除**：只删除认证相关文件（Cookies等），不删除会话数据
- ✅ **选择性恢复**：恢复会话数据时跳过认证文件，使用新账号的认证信息
- ✅ **保留工作环境**：workspaceStorage、History、Session Storage 完整保留

**修改的文件：**
1. `src/main/account-service.ts` - 改进切换账号逻辑
   - 移除 `factoryReset()` 的调用
   - 只调用 `resetMachineId()` 重置机器码
   - 手动删除认证文件，保留会话数据

2. `src/main/backup-service.ts` - 改进恢复逻辑
   - `restoreAll()` 方法新增 `skipAuth` 参数
   - 支持选择性跳过恢复认证文件
   - 切换账号时自动使用新账号的认证

**改进后的切换流程：**
```
1. 备份当前环境（会话、设置、MCP配置）
2. 重置/恢复机器码（不删除会话数据）
3. 清除认证文件（仅 Cookies 等）
4. 执行深度重置（程序文件、DNS）
5. 恢复会话数据（跳过认证文件）
6. 注入新账号 Token
7. 重启 Cursor
```

**用户受益：**
- 🎯 切换账号时会话完整保留
- 🎯 工作区状态不丢失
- 🎯 聊天历史继续可用
- 🎯 用户设置保持不变
- 🎯 MCP 配置继续生效

详细说明请查看：`FIX_SESSION_LOSS.md`

#### 备份管理界面
- 🎨 新增"备份管理"标签页
- 📋 列出所有备份，显示类型、时间、账号

## v1.0.1 (2024-10-28)

### 🔥 新增功能

#### 完整恢复出厂设置
新增**彻底重置Cursor**功能，一键让Cursor回到刚安装的状态！

**清理内容：**
- ✅ 重置机器码（生成全新设备ID）
- ✅ 清除所有认证信息（登出）
- ✅ 清空所有缓存目录（Cache、Code Cache、GPU Cache等）
- ✅ 清空Local Storage
- ✅ 清空Session Storage  
- ✅ 清空工作区历史
- ✅ 清空历史记录
- ✅ 清空备份数据
- ✅ 清空日志文件

**使用场景：**
1. 账号频繁切换被限制，需要彻底重置
2. Cursor出现异常，需要清理缓存
3. 想要完全清除使用痕迹
4. 重新开始使用Cursor

**如何使用：**
1. 在右侧工具面板找到"机器码管理"
2. 点击红色的"🔥 恢复出厂设置"按钮
3. 确认警告提示
4. 等待清理完成
5. 重新启动Cursor，状态将回到全新安装

**注意事项：**
- ⚠️ 这是**危险操作**，会删除所有Cursor数据
- ⚠️ 操作前会自动备份state.vscdb
- ⚠️ 建议先关闭Cursor（工具会自动关闭）
- ⚠️ 恢复后需要重新登录账号

#### 快速导入账号
新增**导入当前账号**按钮，一键从Cursor导入！

**使用方法：**
1. 在Cursor中登录你的账号
2. 在本工具中点击"📥 导入"按钮
3. 输入备注名称（可选）
4. 自动读取并保存token

**优势：**
- 无需手动获取token
- 自动读取邮箱和token
- 快速添加多个账号

### 🔧 修复

- 修复打包后无法启动的问题（路径配置错误）
